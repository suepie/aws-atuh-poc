# B-8: SSO 詳細

> 元データ: [../hearing-checklist.md B-8](../hearing-checklist.md#b-8-sso-詳細-fr-sso-41--proposal-fr-4)  
> 対象: 開発チーム / セキュリティチーム  
> 関連: [proposal §FR-4](../proposal/fr/04-sso.md)

---

### 【クロス IdP SSO 方針（全体方針）】 (B-801, 🟡)

外部 IdP（Auth0 / Entra ID 等）の SSO セッションを本基盤でも信頼するかご教示ください:
- 全顧客有効（業界標準デフォルト）
- 限定（規制業種顧客のみ無効化）
- 無効（毎回再認証）

**業界標準は L1 完全信頼ですが、金融・規制業種では L3 / L4 採用例**もあります。
**目的**: クロス IdP SSO（[§FR-4.2](../proposal/fr/04-sso.md)）の信頼レベル確定。Identity Broker パターン採用時の動作デフォルトを決定する重要な情報です。

---

### 【信頼レベル（L1〜L4）】 (B-801-1, 🟡)

デフォルトの信頼レベルをご教示ください:
- **L1 完全信頼**（業界標準）: IdP セッションあれば即トークン発行、`amr` / `acr` も継承
- **L2 部分信頼**: IdP セッション信頼するが、本基盤側 TTL を別に持つ（短く）
- **L3 検証ありき信頼**: IdP セッション信頼するが、`acr` / `amr` を毎回検査、足りなければステップアップ
- **L4 不信任**: IdP セッションがあっても本基盤側で `prompt=login` で再認証

**目的**: UX とセキュリティのバランス判断、Cognito（`acr_values` / `max_age` 未対応）と Keycloak（標準対応、ただし 26.x の `max_age` バグ注意）の制約への適合性確認に必要な情報です。

---

### 【顧客 / IdP / アプリ別の差別化要否】 (B-801-2, 🟡)

信頼レベルを全顧客で同じにするか、差別化するかご教示ください:
- 全顧客同一の信頼レベル
- 顧客別（例: 金融顧客は L3 / 一般顧客は L1）
- アプリ別（例: 経費精算は L1 / 決済管理は L3）
- 両方（顧客 × アプリ）

**目的**: マルチテナント環境での信頼レベル設計、Authentication Flow の柔軟性要求（Keycloak の優位性）、Cognito 採用時の制約への適合性確認に必要な情報です。

---

### 【規制業種向け L3 オプションの要否】 (B-801-3, 🟢)

金融・医療・PCI DSS 等の規制対応で L3 検証ありき信頼（`acr_values` + ステップアップ）を提供する必要はございますか。
- 必須（規制業種顧客を取り込む）
- 将来検討
- 不要（一般 B2B のみ）

**目的**: プラットフォーム選定への直接影響。**L3 が必須なら Keycloak が事実上必須**（[§C-2.2 A-12](../proposal/common/02-platform.md)）。Cognito では Lambda 自前実装で部分対応可能ですが、AAL 判定とステップアップフローの組み合わせは複雑度高となります。

---

### 【外部 IdP の SSO セッション TTL 尊重】 (B-802, 🟢)

外部 IdP のセッション TTL に従うか、本基盤側で上書きするかご教示ください:
- 従う（業界標準）
- 上書き（本基盤側で短 TTL 強制）
- ハイブリッド: IdP TTL + 短 Access TTL（5-15 分）= 推奨

**目的**: セッション TTL 設計、退職時の即時遮断要件との整合性、`max_age` 制約適用要否の判断に必要な情報です。

---

### 【`max_age` 制約の適用】 (B-802-2, 🟢)

「N 秒以上経過した IdP 認証は再認証要求」する `max_age` 制約を適用しますか。
- 適用する（具体的な秒数 / 分: 例 900 秒 = 15 分）
- 適用しない

**目的**: 古い `auth_time` 悪用対策（[§FR-4.2 リスク 5](../proposal/fr/04-sso.md)）。**Cognito は `max_age` 未サポート、Keycloak は標準対応（ただし 26.x にバグ多数: Issue #32764, #33641, #46078）**のため、採用時はバージョン固定戦略が必要です。

---

### 【同一テナント内 SSO 切断システム】 (B-803, 🟢)

同一テナント内でも SSO を意図的に切りたいシステム（高権限管理画面 / 決済画面等）はございますか。
あり / なし、対象システム名でお答えいただけますと幸いです。
**目的**: 選択的 SSO 制御の要件確定。`prompt=login` の都度付与、Keycloak Authentication Flow Override、Client Session Max の活用設計に必要な情報です。
