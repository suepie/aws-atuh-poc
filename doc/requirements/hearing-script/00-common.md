# Phase A: 事業要件（共通前提）

> 元データ: [../hearing-checklist.md Phase A](../hearing-checklist.md#phase-a-事業要件プロダクトオーナー--事業企画--営業)  
> 対象: プロダクトオーナー / 事業企画 / 営業

---

### 【想定 MAU 規模（1 年後）】 (A-1, 🔥)

1 年後の月次アクティブユーザー数（MAU）の想定値をご教示ください。
具体的な数値（例: 50,000）でお答えいただけますと幸いです。
**目的**: Cognito のティア選定（Lite / Essentials / Plus）と損益分岐点（17.5 万 MAU 前後で Keycloak への切替を検討）の判断に必要な情報です。MAU 規模はプラットフォーム選定（Cognito vs Keycloak）の最重要因子の一つです。

---

### 【想定 MAU 規模（3 年後）】 (A-2, 🔥)

3 年後の MAU 目標値をご教示ください。
具体的な数値でお答えいただけますと幸いです。
**目的**: 中長期のスケーリング設計、マルチリージョン展開要否、Cognito 4,000 万ユーザー / Pool 上限の適合性確認、本基盤の容量設計（Keycloak の Realm Cache チューニング含む）に必要な情報です。

---

### 【対象システム数】 (A-3, 🔥)

共有認証基盤の初期スコープに含めるシステム（経費精算 / 勤怠管理 / 人事システム 等）のリストと優先度をご教示ください。
**目的**: SSO 接続範囲の確定、Cognito App Client 数（1,000 既定 / 10,000 最大）または Keycloak Client 数の見積、リリーススケジュールの策定に必要な情報です。

---

### 【顧客拠点の地理分布】 (A-4, 🔥)

顧客拠点が国内のみか、グローバル展開を想定されているかをご教示ください。
国内のみ / グローバル / 一部海外（地域指定）の形でお答えいただけますと幸いです。
**目的**: マルチリージョン構成の要否、データ所在地（リージョン）の選定、GDPR / CCPA 等のコンプライアンス対応範囲の判断に必要な情報です。

---

### 【既存認証からの移行有無】 (A-5, 🔥)

移行元となる既存認証システムは存在しますか。
存在する場合、対象ユーザー数、現在の認証方式（ローカル / IdP 経由）、保有データ（パスワードハッシュ等）の概要をご教示ください。
**目的**: 移行戦略の策定（段階的移行 / 並行稼働 / ビッグバン）、既存パスワードハッシュの持ち越し要否、ユーザー連続性の設計に必要な情報です。

---

### 【顧客企業の IdP 種別の分布】 (A-6, 🟡)

エンドユーザーである顧客企業の IdP 種別の概算分布をご教示ください。
Microsoft Entra ID / Okta / SAML 系（HENNGE / ADFS 等）/ LDAP / Google Workspace / IdP なし、それぞれの比率と代表的な顧客名（差し支えない範囲で）でお答えいただけますと幸いです。
**目的**: 接続実績のある IdP の種別確認、SAML/LDAP 対応の優先度判断、Keycloak 必須化要因（LDAP 直接連携の有無等）の確認に必要な情報です。

---

### 【新規顧客追加の頻度】 (A-7, 🟡)

月平均で何社の新規顧客追加を想定されているかご教示ください。
**目的**: 顧客追加リードタイム（業界デフォルト < 1 営業日）の目標設定、オンボーディング自動化の必要性、運用工数の見積に必要な情報です。

---

### 【データ所在地要件】 (A-8, 🟡)

ユーザーデータ・認証ログの所在地について、国内限定や特定リージョン制約はございますか。
具体的なリージョン名（東京 / 大阪 / シンガポール 等）でお答えいただけますと幸いです。
**目的**: AWS リージョン選定、Cognito / Keycloak のデプロイ先決定、コンプライアンス要件（金融業の国内データ保管義務等）への適合性確認に必要な情報です。

---

### 【業界規制】 (A-9, 🟡)

顧客の業界における規制要件をご教示ください。
業界名（金融 / 医療 / 政府 / 一般）+ 適用される規制名（PCI DSS / FFIEC / FISC / HIPAA 等）でお答えいただけますと幸いです。
**目的**: パスワードポリシー（NIST SP 800-63B Rev 4 準拠等）、MFA レベル（AAL2/AAL3）、監査ログ保存期間、Cognito Plus ティアや Keycloak RHBK（FIPS 140-2 対応）の必要性判断に必要な情報です。

---

### 【初回リリース時期の目標】 (A-10, 🟢)

初回リリースの目標日をご教示ください。
具体的な日付・年月でお答えいただけますと幸いです。
**目的**: 設計 → 開発 → テスト → 移行の各マイルストーン策定、PoC からの本番移行計画、必要工数の見積に必要な情報です。

---

### 【ブランディング要件（全体方針）】 (A-11, 🟢)

ログイン UI のカスタマイズは必要でしょうか。
必要な場合、対象画面の範囲についてもご教示ください:
- **パターン A**: アプリ画面のみカスタマイズ（ログイン画面は本基盤標準、ログイン後のアプリ画面で完全カスタマイズ）= **業界標準・推奨**
- **パターン B**: 認証基盤画面にも貴社ロゴ表示（Cognito 採用時は 20 顧客上限あり）
- **パターン C**: 完全専用デザイン（Enterprise プラン相当、認証基盤を顧客別に分離）

**目的**: Cognito のティア選定（Essentials+ で Managed Login Branding 利用可）、Keycloak の Theme 開発要否、テナント別の Realm 分離設計、URL allowlist のスコープ判断に必要な情報です。詳細は B-612 で確認します。

---

### 【海外展開時期】 (A-12, 🟢)

海外展開を計画されている場合、その時期と対応すべき法規制をご教示ください。
GDPR（EU） / CCPA（米国カリフォルニア） / その他地域規制 + 時期（または不要）でお答えいただけますと幸いです。
**目的**: マルチリージョン構成の計画、データ越境転送への対応、地域別コンプライアンス機能の優先順位付けに必要な情報です。

---

### 【貴社の社内 IdP】 (A-13, 🟡)

貴社（事業者）が社内で利用されている IdP をご教示ください。
Microsoft Entra ID / Okta / HENNGE / オンプレ Active Directory / なし、のいずれかでお答えいただけますと幸いです。
**目的**: 本基盤の管理アクセス（基盤運用管理者の認証）の設計、Break Glass 用ローカル管理者の必要性判断、γ シナリオ（管理者層のみフェデレーション）採用時の前提確認に必要な情報です。

---

### 【既存 AWS 利用状況】 (A-14, 🟡)

既存の AWS 利用実績についてご教示ください。
利用状況（既存 VPC / 既存 IAM 体制 / Enterprise Support 契約レベル）+ サポート契約レベル（Basic / Developer / Business / Enterprise）でお答えいただけますと幸いです。
**目的**: 既存 AWS インフラへの統合方針、サポートレベルの整合性、コスト見積、運用体制との整合性確認に必要な情報です。
