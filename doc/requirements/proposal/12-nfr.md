## 12.1 可用性（→ NFR-AVL）

### ベースライン（仮）

99.9% / 99.95% / 99.99% のいずれか。マルチ AZ 配置・自動復旧・ゼロダウンデプロイを標準装備。

### TBD / 要確認（仮）

SLA 目標、計画メンテナンス窓

---

## 12.2 性能（→ NFR-PERF）

### ベースライン（仮）

応答時間 P95 < 1s, P99 < 2s。スループットは MAU 規模から逆算。

### TBD / 要確認（仮）

スループット要件、ピーク時間帯倍率

---

## 12.3 拡張性（→ NFR-SCL）

### ベースライン（仮）

MAU 自動スケール、IdP 追加 < 1 営業日、マルチリージョン対応は要件次第。

### TBD / 要確認（仮）

MAU 成長見込み、マルチリージョン要否

---

## 12.4 セキュリティ（→ NFR-SEC §4.1〜§4.4）

### ベースライン（仮）

TLS 1.2+ / KMS / RS256 / Refresh Token Rotation / WAF / Private Subnet / VPC Endpoint。

### TBD / 要確認（仮）

トークン TTL ポリシー、攻撃対策の追加要件（侵害クレデンシャル検出等）

---

## 12.5 DR（→ NFR-DR）

### ベースライン（仮）

RTO/RPO の選定肢を提示。Cognito は Route 53 自動フェイルオーバー、Keycloak は Aurora Global DB。

### TBD / 要確認（仮）

RTO/RPO 目標、DR 訓練頻度

---

## 12.6 運用（→ NFR-OPS §6.1〜§6.3）

### ベースライン（仮）

CloudWatch 監視、CI/CD 自動デプロイ、24/7 体制は要件次第。

### TBD / 要確認（仮）

ログ保存期間、24/7 サポート要否、運用主体

## 12.7 コンプライアンス（→ NFR-COMP §7.1〜§7.3）

### ベースライン（仮）

個人情報保護法 Must。GDPR / SOC 2 / ISO 27001 / FIPS は要件次第。

### TBD / 要確認（仮）

適用される業界規制、データ所在地制約

## 12.8 コスト（→ NFR-COST）

### ベースライン（仮）

MAU 規模次第で Cognito vs Keycloak が損益分岐。3 年 TCO で比較提示。

### TBD / 要確認（仮）

予算レンジ、コスト評価期間

---

## 12.9 移行性（→ NFR-MIG）

### ベースライン（仮）

既存認証システムからの段階的移行をサポート。パスワードハッシュ移行は方式次第。

### TBD / 要確認（仮）

既存システム種別、移行期限
