## §7.0 前提と背景

### §7.0.1 用語整理

| 用語 | 定義 |
|---|---|
| **PII** | Personally Identifiable Information（個人識別情報） |
| **業界規制** | PCI DSS / FFIEC / FISC / HIPAA / GDPR / CCPA / 個人情報保護法 |
| **データガバナンス** | データの分類・所在・保持期間・廃棄の管理 |
| **監査ログ** | 規制対応・内部統制のために保管する記録（CloudTrail 等） |

### §7.0.2 なぜここ（§7）で決めるか

API プラットフォームは複数アプリの共通基盤として、各アプリのコンプラ要件を満たせる前提で標準を定める。**個別アプリの特殊要件は本標準を超える追加対応**として例外承認制（§FR-7 §7.4）にする。

### §7.0.3 IPA グレード対応

| IPA 中項目 | 本章での対応 |
|---|---|
| 規制 / 法令 | §7.1 |
| 業界認定 / 監査 | §7.2 |
| データガバナンス | §7.3 |

### §7.0.4 §7.0.A 本標準のスタンス

| 基本方針 | 本章での具体化 |
|---|---|
| 絶対安全 | 規制対応をデフォルトのガードレールに織り込む（事後対応を避ける） |
| どんなアプリでも | 業界別の追加要件をアドオンとして整理 |
| 効率よく | AWS の各種認定（PCI / SOC 2 / ISMS 等）を活用 |
| 運用負荷・コスト最小 | Security Hub の標準（CIS / AWS FSBP / PCI DSS）で継続評価 |

## §7.1 規制対応

**このサブセクションで定めること**：本標準が想定する規制範囲とアドオン要件。
**主な判断軸**：適用される規制、AWS の認定範囲。
**§7 全体との関係**：§NFR-4 セキュリティ死守事項と整合。

### §7.1.1 ベースライン

| 規制 | 既定対応 | アプリ別アドオン |
|---|---|---|
| **個人情報保護法（日本）** | PII マスキング・暗号化・所在地国内 | 第三者提供時の同意管理 |
| **GDPR（EU）** | データ削除権 / 越境転送 SCC | アプリ側で削除フロー |
| **CCPA（カリフォルニア）** | データ削除権 | アプリ側で削除フロー |
| **PCI DSS（カード決済）** | Cognito / Keycloak / Lambda は対応外、Vault 経由 | 決済 API は別アーキ |
| **HIPAA（医療）** | BAA 締結後の対象サービスのみ | 医療 API は別アカウント |
| **FISC（金融）** | リージョン・暗号化・監査 | 金融 API は別アカウント |

## §7.2 業界認定・監査

**このサブセクションで定めること**：監査・認定の対応標準。
**主な判断軸**：必要な認定（SOC 2 / ISMS / PrivacyMark / ISO 27001）と AWS の認定活用。
**§7 全体との関係**：監査エビデンス収集の標準。

### §7.2.1 ベースライン

- **AWS Artifact** で AWS の認定レポートを取得（SOC 2 / ISO 27001 / PCI DSS 等）
- **AWS Audit Manager** で監査エビデンス自動収集
- **CloudTrail Org trail / Config aggregator** で証跡保管
- **保管期間**：7 年（規制業種）/ 3 年（一般）

### §7.2.2 TBD / 要確認

- Q: 必要な **認定リスト**（SOC 2 Type II / ISMS / PrivacyMark / ISO 27001）→ `API-D-1511`
- Q: Audit Manager の **採用範囲** → `API-D-1512`

## §7.3 データガバナンス

**このサブセクションで定めること**：データの分類・保持・廃棄。
**主な判断軸**：DataClassification タグ、保持期間明示、廃棄手段。
**§7 全体との関係**：§FR-4 必須タグの根拠。

### §7.3.1 ベースライン

- **DataClassification タグ**（必須）：`public` / `internal` / `confidential` / `pii`
- **保持期間**：
  - PII：必要最小限、業務終了後 N 年で削除
  - ログ：§FR-8 §8.1 のリテンション規約
  - 監査ログ：7 年
- **廃棄**：S3 Lifecycle、自動削除
- **データマスキング**：開発・ステージング環境では PII マスク

### §7.3.2 TBD / 要確認

- Q: PII の **業務別保持期間**確定 → `API-D-1521`
- Q: 開発環境の **データマスキング手段**（DMS / Glue 等）→ `API-D-1522`

## §7.4 データ越境転送

**このサブセクションで定めること**：海外リージョン利用時の規制対応。
**主な判断軸**：データ所在地、SCC / 同意。
**§7 全体との関係**：§NFR-5 DR のマルチリージョンと相反するケースあり。

### §7.4.1 ベースライン

- **データ所在地**：原則 ap-northeast-1（東京）または ap-northeast-3（大阪）
- **海外リージョン利用は例外承認制**：理由 / SCC / 同意取得を明示
- **GDPR 対象データの EU 域外転送**：SCC + 暗号化
