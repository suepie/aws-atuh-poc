# ADR-001: PoC第1パターンとしてCognitoハイブリッド構成を採用

**ステータス**: Accepted
**日付**: 2026-03-17

## コンテキスト

統合認証基盤としてCognitoとKeycloakのどちらが適切か判断するため、PoCを実施する。
まずCognitoのハイブリッド構成（集約Cognito + ローカルCognito）を第1パターンとして構築する。

## 決定

PoC第1パターンとして **Cognitoハイブリッド構成** を採用する。

### 構成
- 集約Cognito: 外部IdP（Auth0 Free）によるフェデレーション認証
- ローカルCognito: パートナーユーザー等のローカル認証
- Lambda Authorizer: マルチissuer対応JWT検証

## 理由

1. 過去の検討ドキュメントでCognitoが推奨されており、まず推奨案の実現性を検証する
2. ハイブリッド構成はSSO + 障害分離を両立する設計で、本番想定に最も近い
3. Keycloakは第2パターンとして後日構築し、比較材料とする

## 代替案

| 代替案 | 不採用理由 |
|-------|-----------|
| Keycloakから先に試す | Cognito推奨の検討結果があり、まず推奨案を検証する方が効率的 |
| 集約型のみ（ローカルCognitoなし） | ハイブリッド構成のマルチissuer対応を検証できない |
| 分散型のみ（集約Cognitoなし） | SSO検証ができない |

## 影響

- Phase 1〜4でCognito構成を段階的に構築
- Phase 6でKeycloak構成を構築し比較
- 最終判断はPoC完了後
