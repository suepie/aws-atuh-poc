# ADR-006: Cognito vs Keycloak コスト損益分岐点の分析

**ステータス**: Proposed（PoCの結果を踏まえて最終判断）
**日付**: 2026-03-17
**最終更新**: 2026-03-17

## コンテキスト

顧客はコストメリットからKeycloakを視野に入れている。開発チームは運用負荷の観点からCognitoを推したい。判断にはMAU規模に基づくコスト比較が必要。

## 分析

### 前提条件

- Cognito料金: 2024年11月改定後の3ティア制（Lite/Essentials/Plus）
- フェデレーション（OIDC/SAML）: **全ティア共通 $0.015/MAU**（ボリューム割引なし）
- フェデレーション無料枠: **50 MAU/月**（アカウントまたはOrganization単位）
- Keycloak: AWS上のHA構成（ECS Fargate + Aurora PostgreSQL Multi-AZ）
- 運用人件費: エンジニア単価 $80/h、月21時間

### Cognito 料金体系（2024年11月改定）

#### 直接サインイン・ソーシャルIdPユーザー（Liteティア）

| MAU範囲 | 単価 |
|---------|------|
| 0 - 10,000 | 無料 |
| 10,001 - 100,000 | $0.0055/MAU |
| 100,001 - 1,000,000 | $0.0046/MAU |
| 1,000,001 - 10,000,000 | $0.00325/MAU |

#### SAML/OIDCフェデレーション（全ティア共通）

| MAU範囲 | 単価 |
|---------|------|
| 0 - 50 | 無料 |
| 51以降 | **$0.015/MAU（一律）** |

**重要: フェデレーションにはボリューム割引がない。これがコスト比較の鍵。**

### Keycloak 固定コスト（HA構成）

| コンポーネント | 月額 |
|--------------|------|
| ECS Fargate（4vCPU × 8GB × 3タスク） | $510 |
| Aurora PostgreSQL Multi-AZ（db.r6g.large × 2） | $300 |
| ALB | $50 |
| NAT Gateway（2 AZ） | $80 |
| CloudWatch + Secrets Manager | $60 |
| **インフラ小計** | **$940/月** |
| 運用人件費（月21h × $80/h） | **$1,680/月** |
| **合計** | **$2,620/月** |

### 損益分岐点

```
Cognito月額 = Keycloak月額
(MAU - 50) × $0.015 = $2,620
MAU ≈ 175,000
```

### 損益分岐点に影響する変動要因

| 要因 | 分岐点への影響 | 説明 |
|------|-------------|------|
| 運用人件費の上昇 | 分岐点が上がる（Cognito有利に） | Keycloak障害対応で想定以上の工数 |
| Keycloakスケールアップ | 分岐点が上がる（Cognito有利に） | 50万MAU超でインフラ増強が必要 |
| Cognito値下げ | 分岐点が上がる（Cognito有利に） | AWS値下げの可能性 |
| Keycloak Managed Service利用 | 分岐点が下がる（Keycloak有利に） | Phase Two等で運用負荷低減 |

## コスト以外の比較

| 観点 | Cognito | Keycloak |
|------|---------|----------|
| 初期構築費 | $5,000 | $30,000 |
| DR対応 | Route 53フェイルオーバー（容易） | 公式非推奨（自前実装） |
| セキュリティパッチ | AWS自動適用 | 自己責任（CVE対応工数） |
| SLA | 99.9%保証 | 保証なし（自己責任） |
| スケーリング | 自動 | 手動設計 |
| カスタマイズ性 | 制限あり | 高い |
| ベンダーロックイン | AWS依存 | OSSで自由 |

## 暫定結論

**17.5万MAU未満 → Cognito推奨**
- コスト・運用・DR・セキュリティすべてにおいてCognitoが有利
- 初期構築費の差（$25,000）も大きい

**17.5万MAU以上 → 要検討**
- コストのみKeycloakが有利になるが、運用リスクと人件費変動を加味すると実質的な分岐点はさらに上がる
- 特にDR要件がある場合はKeycloakの対応が困難
