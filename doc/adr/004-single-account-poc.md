# ADR-004: 1アカウント2 User Poolでマルチアカウント構成を擬似再現

**ステータス**: Accepted
**日付**: 2026-03-17

## コンテキスト

本番想定はマルチアカウント構成（共通認証基盤アカウント + 各サービスアカウント）だが、
PoCで使えるAWSアカウントは1つのみ。

## 決定

1つのAWSアカウント内に **2つのCognito User Pool** を作成し、マルチアカウント構成を擬似再現する。

- **User Pool A**: 集約Cognito（共通認証基盤アカウント相当）
- **User Pool B**: ローカルCognito（サービスアカウント相当）

## 理由

1. **JWKS取得はHTTPS公開エンドポイント** → アカウントをまたがってもIAM設定不要
2. issuerはUser Pool IDで決まる → 同一アカウントでもissuerは異なる
3. Lambda Authorizerのマルチissuer判定ロジックはそのまま検証可能
4. 本番移行時は環境変数（User Pool ID, Client ID）を変更するだけ

## 差異（本番との違い）

| 観点 | 本番 | PoC | 影響 |
|------|------|-----|------|
| アカウント分離 | 別アカウント | 同一アカウント | IAM権限分離の検証不可 |
| ネットワーク | クロスアカウント | 同一VPC | レイテンシ差異なし |
| JWKS取得 | HTTPS（同じ） | HTTPS（同じ） | **差異なし** |
| Lambda Authorizer | サービスアカウント | 同一アカウント | ロジックは同一 |
| コスト分離 | アカウント別 | 不可 | PoCでは不要 |

## 結論

認証フローの検証には十分。アカウント分離・権限分離の検証は本番環境で別途行う。
